自序

作为一个自动化专业的本科生我已毕业快五年了，这五年时间里我尝试学习过很多专业相关的和不相关的东西，其实，自动化专业的课程本来就很杂，所以这些年涉及各种学科也不算偏离本职。处处涉猎然而处处不精，这似乎是这些年学习各种知识的总结。09年毕业，玩了一年时间的个人博客渐渐因为工作的原因疏于更新，由于博客系统搭建在wordpress平台之上，搭建更新博客的这段时间也算对PHP多少有些了解，虽然这段过程随着博客的关闭没有留下点滴，但是写博客的过程养成了我随手记录的习惯，时至今日不得不感谢当年那一段时间的馈赠。
有人说毕业后五年这段时间能决定人的一生，虽然我没有明显感觉到这段时间对我有多少影响，但是五年后的今天我唯一能感触到的是：我将注定对这段时间后悔一生！五年时间我先后学习过51单片机、ARM10、C++、QT、Python、Delphi，很惭愧的是虽然学习过很多知识，但是这些知识无一不是浅尝则止，无一而终。记得当时学习Python的目的是为了编写一款基于数据库的仓库管理软件，在将软件做到成型的时候发现Python编写的软件打包成.exe文件太麻烦而转投Delphi，唯一感到欣慰的是，这款用Delphi编写的“Qotus 仓库备件管理软件”至今能在单位使用，这已经是2013年的事情了。之后由于在车间一些设备上看到了无处不再的AVR单片机开始关注并转投AVR单片机学习，在学习过程中AVR作为唯一一款被功能强大的GCC编译器支持的8位单片机其丰富的单片机外围资源无不让我欣喜若狂，再加上对开源硬件的逐步了解，有幸认识了Arduino 这个基于AVR单片机的硬件模块，积木式的开发板资源，在开源的世界，我被一次次激发了思维，学习已经不再孤独，因为在开源的世界你不是一个人在战斗！

开发软件配置

工欲善，必先利其器。

现在的集成开发软件有很多种，这里推荐使用CodeBlocks，当然，使用什么开发软件都无所谓，关键是自己熟练。

CodeBlocks配置

CodeBlocks可以很方便的进行多种开发环境的设置，由于是C++开发的，效率个人觉得都很不错，本教程探讨的是软件具体配置，所以不讲解软件的安装。本教程所有操作是在CodeBlocks 13.10 版进行，若是其他版本稍作修改应该也能适用。

设置GNU GCC Compier环境

  1. 打开软件，在主菜单中依次选择：Settings -> Compiler 。
    设置窗口截图
    如图，选择“GNU AVR GCC Compiler”编译器。
  2. 选取“Toolchain executables”选项卡，此处即为设置系统所安装的AVR-gcc编译工具链的环境路径设置。
    AVR GCC工具链环境设置
    如图，此处设置“Compiler’s installation directory”，也就是设置包含“bin”文件夹的目录。
  3. 库文件目录设置。
    选取“Search directories”选项卡，点击左下脚的“Add”按钮，选择包含avr-gcc中的库文件所在目录，如下图所示：

利用Arduino软件编译optiboot bootloader

固化bootloader前的准备——设置芯片熔丝位

用proisp或者同等软件借助USBASP将芯片熔丝位设置成如下：

  1. HFUSE = DE
  2. LFUSE = FF
  3. EFUSE = FD
  4. 加密位最好别动

编译Optiboot并固化芯片

一直在用Sanguino的bootloader，这个已经编译好的bootloader虽然很大（4KB），但是对于Qotusun-mini的Atmega644pa主控来说还算能接受，最近新作了几块新主控板，需要往里下载bootloader，由于Microduino也是用的Atmega644p作为其Core+的主控芯片，而且发现其用的是Optiboot改过来的，都知道Optiboot是出了名的精简，所以想偷懒直接用Microduino生成好的bootloader，谁知下进去之后发现AVRDUDE将主控识别为Atmega644，虽然只是下载程序用，并不影响其他，但是强迫症一直让我对其耿耿于怀，必须将其纠正才是！看了看Optiboot自带的源码，makefile文件中居然有Sanguino的编译选项，so,why not!发现最简单的编译方法就是在Arduino hardware文件夹中用自带的批处理进行自动编译，oh ,my god!,so easy!!!于是将Microduino固件直接放在Arduino软件中，在Windows中直接进入CMD命令行，进入Microduino/bootloaders/optiboot目录:

然后键入指令并运行：

于是一个名叫optiboot_atmega644p的intel hex文件生成了，这个就是我们需要的bootloader文件，看看大小只有1.5Kb，然后利用USBASP固化进芯片中，用AVRDUDE检测芯片，“ATmega644p”！恭喜你AVRDUDE君，你答对了！用Qotusun usb下载模块连上主控模块将测试程序下载后进行验证，Bingo!就这么简单，妈妈从此再也不用担心我得学习了，因为以后可以告别专门的下载器而使用Qotusun usb下载模块进行程序下载和串口调试了！
